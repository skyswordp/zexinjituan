# 项目提交历史总结（本地仓库视图）

## 分支概况
- 三个仓均以 `master` 为主干；`png` 额外有远端 `acea-dev`，未见长期活跃。

## 提交历史与频率
- **png**：2019 年起至今约 2,494 次提交，覆盖 1,024 个日期，平均约 2.4 次/天（高频日更）。近几年以 NN\Nice 高频维护，最近有 SREM 调整 Dockerfile。
- **crm-server**：2018-08 起至今约 1,291 次提交，覆盖 486 个日期，平均约 2.7 次/活跃日。早期 Vigoss 初始化，近期有 SREM Dockerfile 更新。
- **crm-vue**：2018-08 起至今约 410 次提交，覆盖 209 个日期，平均约 2.7 次/活跃日。早期 Vigoss 初始化，近期 NN\Nice 负责补充/格式调整。

## 核心工作内容（基于提交信息归纳）
- **png（采集/清洗层）**：多厂商体育/真人/电子数据抓取接入与补录，定时任务（quartz）调整，日志与去重优化，线程池/性能参数调优，偶发 FTP/SQL/配置修正，近期 Dockerfile 构建调整。
- **crm-server（后端）**：CRM/风控接口与定时任务迭代，构建/配置调整（含 Dockerfile），RB 等产品相关更新。
- **crm-vue（前端）**：CRM/风控后台页面的功能补充、格式与缺失项修复。

## 维护建议
- 开发直接在 `master`（如需特殊场景，`png` 可关注远端 `acea-dev`）。
- 避免提交生产敏感配置；本地 dev 配置与生产配置分离，生产配置由运维/SRE 托管。
- 表结构/配置变更需先确认需求并与 DBA/SRE 同步。
